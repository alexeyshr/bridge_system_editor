# ТЗ: BridgeHub — Портал для бриджевого сообщества

> Вдохновлено LessWrong.com — минималистичная платформа для контента, обучения и аналитики в спортивном бридже.

---

## 1. Концепция

**BridgeHub** — это портал для русскоязычного бриджевого сообщества, объединяющий:
- 📝 Авторский контент (статьи, разборы, обучение)
- 📊 Аналитику турниров и рейтингов
- 🎓 Обучающие серии и библиотеку
- 🃏 Интерактивные разборы раздач
- 👥 Профили игроков и пар

**Целевая аудитория:**
- Игроки в спортивный бридж (от начинающих до экспертов)
- Тренеры и авторы обучающих материалов
- Организаторы турниров
- Судьи

**Позиционирование:** «LessWrong для бриджа» — фокус на качестве контента, репутации авторов, структурированном знании.

---

## 2. Ключевые сущности

### 2.1 Пользователи

#### Доступ к порталу

Портал **открыт для всех** — незарегистрированные посетители могут свободно читать контент. Это важно для **популяризации бриджа** в России: случайный человек может зайти, прочитать статью и заинтересоваться. Регистрация требуется для активного участия (комменты, голоса, отзывы, публикации).

#### Роли (технические)

Всего **две роли** на портале:

| Роль | Описание |
|------|----------|
| 🃏 **Игрок** | Базовая роль. Присваивается автоматически при регистрации. Любой зарегистрированный пользователь — Игрок. |
| 🛡️ **Модератор** | Служебная роль. Назначается администрацией. Управление контентом, арбитраж отзывов, выдача/отзыв привилегий. |

#### Привилегии

Привилегии — это **дополнительные возможности**, которые открываются для Игрока. Совмещаются свободно. **Не привязаны к карме** — выдаются по отдельным критериям (верификация, заявка, решение модератора).

| Привилегия | Как получить | Что открывает |
|------------|-------------|---------------|
| 📝 **Автор** | По заявке или автоматически (критерии TBD) | Публикация постов, статей, серий |
| 🎓 **Тренер** | Верификация (квалификация, опыт, отзывы) | Создание курсов (платных/бесплатных), проведение занятий, ученики |
| ⚖️ **Судья** | Верификация (сертификат ФСБР, категория) | Отображение в турнирах как судья, получение отзывов как судья |
| 🏆 **Организатор** | Верификация (проведён ≥1 турнир) | Создание/управление турнирами на портале, привязка площадок |

> ℹ️ Совмещение — норма. Один человек может быть Игроком + Автором + Судьёй + Организатором одновременно (как часто бывает в российском бридже).

### 2.2 Посты (статьи)

| Поле | Описание |
|------|----------|
| Заголовок | Название статьи |
| Автор(ы) | Один или несколько авторов |
| Тело | Markdown + расширения (раздачи, торговля, диаграммы) |
| Теги | Множественные теги-концепции |
| Тип | article / analysis / tutorial / review / news |
| Серия | Принадлежность к серии (Sequence) |
| Статус | draft / published / curated / archived |
| Karma | Сумма голосов |
| Дата публикации | |
| Связанные раздачи | Ссылки на интерактивные раздачи |
| Сложность | Beginner / Intermediate / Advanced / Expert |

**Типы контента:**
- **Статья (article)** — авторский текст на любую тему бриджа
- **Разбор (analysis)** — анализ конкретного турнира / матча / раздачи
- **Учебный материал (tutorial)** — обучающая статья с примерами
- **Обзор (review)** — обзор книги, приложения, ресурса
- **Новость (news)** — событие в мире бриджа

### 2.3 Quick Takes (Заметки)

Короткие публикации (до 1000 символов):
- Интересная раздача дня
- Наблюдение с турнира
- Вопрос сообществу
- Быстрый совет

### 2.4 Комментарии

- Древовидная структура (как Reddit / LessWrong)
- Голосование (karma)
- Реакции: 👍 Согласен / 👎 Не согласен / 💡 Инсайт / 🤔 Спорно
- Возможность цитировать часть поста

### 2.5 Теги (Концепции)

Каждый тег — это **вики-страница**:

```
Тег: Stayman
├── Определение: Конвенция Стейман — ответ 2♣ на открытие 1NT...
├── Варианты: Puppet Stayman, Garbage Stayman, Extended...
├── Связанные теги: [Transfers, 1NT Opening, Conventions]
├── Статьи с этим тегом: [...]
└── Примеры раздач: [...]
```

**Категории тегов:**
- **Конвенции:** Stayman, Blackwood, Transfers, Michaels, etc.
- **Техника розыгрыша:** Сквиз, Эндплей, Крос-рафф, Элиминация
- **Торговля:** Форсинг, Инвитационные, Конкурентные
- **Защита:** Сигналы, Стартовые выходы, Контратака
- **Форматы:** IMP, MP, BAM, Швейцарка, Раунд-робин
- **Общее:** Психология, Этика, История, Правила

### 2.6 Серии (Sequences)

Упорядоченная коллекция постов, как главы книги:

Примеры серий:
- «Основы системы 2/1 Game Forcing» (15 частей)
- «Защита для продвинутых» (10 частей)
- «Сквиз-техника: от простого к сложному» (8 частей)
- «Психология за столом» (5 частей)
- «Как начать играть в бридж» (20 частей, для новичков)

### 2.7 Раздачи (Deals)

Интерактивный компонент — ключевое отличие от LessWrong.

| Поле | Описание |
|------|----------|
| board_number | Номер сдачи |
| dealer | Раздающий (N/S/E/W) |
| vulnerability | Зона (None/NS/EW/Both) |
| hands | 4 руки (52 карты) |
| bidding | Последовательность торговли |
| play | Последовательность розыгрыша (опционально) |
| contract | Контракт |
| result | Результат (взятки) |
| analysis_text | Текстовый комментарий |
| source_tournament | Ссылка на турнир |
| tags | Теги (sквиз, эндплей, etc.) |
| difficulty | Сложность |

**Визуализация:**
- Диаграмма руки (N/S/E/W)
- Пошаговый разбор торговли с комментариями
- Пошаговый розыгрыш с подсветкой ключевых моментов
- Double Dummy анализ
- Сравнение результатов (IMP/MP таблица)

### 2.8 Турниры

Интеграция с bridgesport.ru:
- Календарь предстоящих турниров
- Результаты прошедших
- Рейтинги игроков
- Статистика пар

### 2.9 Клубы

Бриджевые клубы — важная организационная единица сообщества, обычно привязаны к городу.

| Поле | Описание |
|------|----------|
| Название | «Московский бриджевый клуб», «Клуб "Бридж в Питере"» |
| Город | Город расположения |
| Описание | О клубе, история, направленность |
| Площадка | Основная площадка клуба (связь с Venue) |
| Руководитель | Пользователь-контакт |
| Участники | Список игроков-членов клуба |
| Расписание | Регулярные игры, занятия |
| Соцсети | Telegram-группа, VK, сайт |
| Логотип | Изображение |

**Функции клуба на портале:**
- Страница клуба с информацией и списком участников
- Клубные турниры и результаты
- Клубный рейтинг (агрегат рейтингов участников)
- Лента новостей клуба
- Игрок может состоять в **одном или нескольких** клубах

### 2.10 Обучение (Тренер ↔ Ученик)

Связь «тренер — ученик» — ключевая для образовательной части портала.

| Поле | Описание |
|------|----------|
| Тренер | Пользователь с привилегией «Тренер» |
| Ученик | Любой зарегистрированный Игрок |
| Статус | active / completed / paused |
| Начало | Дата начала обучения |
| Курс | Привязка к конкретному курсу (опционально) |

**Возможности тренера:**
- Создание курсов (серия уроков с материалами, задачами, домашками)
- Платные и бесплатные курсы
- Отслеживание прогресса учеников
- Получение отзывов от учеников → влияет на репутацию тренера

**Возможности ученика:**
- Запись на курсы
- Прогресс и достижения
- Оценка тренера по завершении курса
- Рекомендации тренера другим игрокам

### 2.11 Команды

Команда — сущность для командного бриджа (4+ игрока).

| Поле | Описание |
|------|----------|
| Название | «Сборная Москвы», «Dream Team» |
| Капитан | Пользователь-контакт |
| Состав | Список игроков (4-6, с возможностью замен) |
| Клуб | Привязка к клубу (опционально) |
| Командный рейтинг | Агрегат рейтингов участников |
| История | Результаты командных турниров |

**Функции:**
- Страница команды с составом и историей
- Управление составом (ротация, замены)
- Командные турниры и результаты
- Командный рейтинг

### 2.13 Bidding Polls (голосования по торговле)

Ключевой формат контента, генерирующий максимум вовлечения (проверено BridgeWinners).

| Поле | Описание |
|------|----------|
| Рука | 13 карт (рука голосующего) |
| Контекст | Зона, позиция, предыдущая торговля |
| Варианты заявок | Автоматически + «другое» |
| Голоса | Распределение голосов по заявкам |
| Комментарии экспертов | Разбор после закрытия голосования |
| Срок | Дата закрытия голосования |

**Механика:**
- Любой Автор может создать Bidding Poll
- Голосование анонимное до закрытия
- После закрытия: статистика по уровням игроков («как голосуют новички vs эксперты»)
- Еженедельный Bidding Poll — регулярная рубрика
- Связь с задачником: опубликованные поллы становятся задачами

### 2.14 Конвенционные карты (Convention Cards)

Онлайн-редактор и хранилище конвенционных карт.

| Поле | Описание |
|------|----------|
| Пара | Привязка к паре (partnership) |
| Шаблон | WBF стандарт / ACBL / кастомный |
| Версия | Номер версии (история изменений) |
| Содержимое | Структурированные данные системы |
| QR-код | Автогенерируемый для показа оппонентам |

**Функции:**
- Совместное онлайн-редактирование (оба партнёра)
- Версионирование: история изменений, привязка версии к турниру
- Печать в формате WBF (PDF)
- QR-код → ссылка на онлайн-версию
- Хранилище всех КК с разными партнёрами

### 3.2 Лента и фильтрация

**Режимы ленты:**
- **Свежее** — хронологически, все посты
- **Лучшее** — отсортировано по karma за период
- **Курированное** — отобрано редакцией
- **Персональное** — на основе подписок и интересов

**Фильтры по тегам (как LessWrong):**
- При наведении на тег: слайдер «больше/меньше» этого контента
- Сохранение персональных фильтров

**Фильтры по сложности:**
- 🟢 Начинающий
- 🟡 Средний
- 🟠 Продвинутый
- 🔴 Эксперт

### 3.4 Редактор контента

**Markdown+** — расширенный Markdown:

Стандартный Markdown +
- `:::deal` — блок с раздачей (парсит PBN/LIN формат)
- `:::bidding` — таблица торговли с комментариями
- `:::play` — пошаговый розыгрыш
- `:::quiz` — интерактивный вопрос (выбор хода/заявки)
- `:::diagram` — диаграмма руки

Пример:
```markdown
Посмотрим на эту раздачу:

:::deal
[Board "15"]
[Dealer "N"]
[Vulnerable "EW"]
[N "AK742.Q3.KJ5.A82"]
[E "983.1074.10973.Q6"]
[S "65.AK62.A62.J1073"]
[W "QJ10.J985.Q84.K95"]
:::

Торговля была следующей:

:::bidding
N: 1♠ — Натуральное открытие
S: 2♥ — 2/1 GF, 5+ червей (нет, 4 черв + поддержка пик)
N: 3♦ — Доп. информация
S: 4♠ — Финальный контракт
:::

Ключевой момент на 7-м ходе:

:::quiz
type: lead
hand: S
question: "Какой ваш ход?"
options:
  - "♥A — собрать козырь": wrong, "Упустите контроль"
  - "♦6 — к столу": correct, "Готовим сквиз!"
  - "♣J — атака": wrong, "Рано для клубов"
:::
```

### 3.9 Система репутации (Karma 2.0)

#### 3.9.1 Структура кармы

Карма пользователя состоит из **двух независимых осей**:

| Ось | Что измеряет | Из чего складывается |
|-----|-------------|---------------------|
| 📝 **Контент-карма** | Вклад в портал | Голоса за посты, комменты, серии, вклад в вики, обучающие материалы |
| 🤝 **Репутация** | Поведение в сообществе | Пост-турнирные отзывы (как игрок, судья, организатор, тренер) |

Обе оси отображаются в профиле **раздельно** — это разные вещи. Человек может писать отличные статьи, но хамить за столом (или наоборот).

**Общая карма** = контент-карма + репутация (для агрегированного отображения).

> ⚠️ Привилегии (Автор, Тренер, Судья, Организатор) **не привязаны к карме**. Они выдаются по отдельным критериям (верификация, заявка, решение модератора). Зависимость привилегий от кармы — возможна в будущем, но не на старте.

#### 3.9.2 Контент-карма

Начисляется любому пользователю за вклад в контент портала.

**Голосование за контент:**
- +1/-1 за голос обычного пользователя
- +2/-2 за голос пользователя с общей karma > 100
- +3/-3 за голос пользователя с общей karma > 500
- Отдельно: количество «Agree» и «Disagree» (как LessWrong)

**Реакции на комменты:**
- 👍 Согласен / 👎 Не согласен — не влияют на karma
- ⬆ Полезно / ⬇ Не полезно — влияют на karma
- 💡 Инсайт — бонус +2 к karma автора
- 🏆 Отличный анализ — бонус +5 (только для karma > 200)

**Вклад в сообщество** (бонусная карма):
- Создание/редактирование тегов-концепций (вики)
- Модерация контента
- Ответы на вопросы новичков
- Перевод материалов

#### 3.9.3 Репутация (офлайн-отзывы)

Применяется ко всем: как **игрок за столом**, так и как **судья, организатор, тренер** — через соответствующие отзывы.

Собирается через **пост-турнирные отзывы** (см. раздел 3.10).

**Категории оценки игрока:**
- **Этичность** — соблюдение правил, отсутствие нарушений (несанкционированная информация, замедление темпа)
- **Корректность** — вежливость, уважение к оппонентам и партнёру
- **Пунктуальность** — приход вовремя, соблюдение темпа игры

Отзыв оставляется **только после турнира**, в котором оба игрока участвовали. Анонимный для публики, но видим модераторам.

#### 3.9.4 Пост-турнирная обратная связь

> После завершения каждого турнира, зарегистрированного на портале, участники получают приглашение оставить обратную связь.

**Сбор отзывов:** без жёсткого дедлайна, оставить отзыв можно в любое время.
- 📩 **Напоминание** через 24 часа после турнира: «Как прошёл турнир? Оставьте отзыв»
- ⏱️ **Поздний отзыв** (> 7 дней после турнира): пометка «поздний», вес на карму снижен ×0.5
- 🎯 Цель: стимулировать быструю обратную связь, но не блокировать позднюю

**Три блока отзывов:**

**1. 🏆 Организация и судейство**
- Оценка организации (1-5 ⭐): подготовка, расписание, логистика, информирование
- Оценка судейства (1-5 ⭐): компетентность, объективность, оперативность решений
- Текстовый комментарий (опционально)
- → Влияет на **Karma-Organizer** и **Karma-Judge** соответствующих людей

**2. 🃏 Поведение игроков (замечания)**
- Заполняется **только при наличии замечаний** (не обязательный блок)
- Выбор игрока из списка участников турнира
- Категория: этичность / корректность / пунктуальность / другое
- Описание ситуации (обязательный текстовый комментарий)
- Тон: конструктивная обратная связь, не жалоба
- → Влияет на **Karma-Player** указанного игрока (после модерации!)

**3. 📍 Площадка и место проведения**
- Оценка площадки (1-5 ⭐): удобство, освещение, вентиляция, шум
- Оценка доступности: транспорт, парковка, навигация
- Оценка удобств: напитки/еда, туалеты, зона отдыха
- Что понравилось (текст)
- Что можно улучшить (текст)
- → Привязывается к **сущности «Площадка»** (venue) — копится рейтинг площадок

**Визуализация результатов:**
- На странице турнира: сводка отзывов (средние оценки + облако комментов)
- На профиле организатора/судьи: агрегированные оценки по всем турнирам
- На странице площадки: рейтинг и отзывы

#### 3.9.5 Защита от злоупотреблений

Карма — чувствительная тема. Система должна быть **справедливой и устойчивой к манипуляциям**:

**Анти-абьюз для офлайн-отзывов:**
- ⚖️ **Модерация:** негативные отзывы об игроках проходят модерацию перед влиянием на карму
- 🔢 **Порог значимости:** минимум 3 отзыва от разных людей, чтобы офлайн-карма начала меняться
- ⚖️ **Вес отзыва** зависит от кармы отзывающегося (высокая карма = больший вес)
- 🔄 **Детекция взаимного занижения:** если A→B и B→A оба негативные — флаг на модерацию
- 📊 **Детекция аномалий:** массовое занижение одним пользователем или координированная атака
- ⏱️ **Rate limiting:** максимум 5 негативных отзывов об игроках за один турнир
- 🗣️ **Апелляция:** игрок может оспорить отзыв, решает модератор/арбитр

**Анти-абьюз для онлайн-кармы:**
- Голос нового аккаунта (< 7 дней) весит ×0.5
- Нельзя голосовать за свои посты/комменты
- Массовое голосование одному автору за короткий период — автофлаг

### 3.11 Поиск партнёра и маркетплейс

**Статус игрока в профиле:**

Система рекомендаций: подбор партнёра по рейтингу, городу, системе, расписанию, истории совместных игр.

#### 3.11.3 Маркетплейс платных партнёров

Профессионалы выставляют предложения: «Играю за N руб/турнир». Профиль с рейтингом, отзывами, статистикой. Фильтрация по рейтингу, городу, формату, цене. Оплата через кошелёк с эскроу (деньги замораживаются до завершения турнира).

### 3.13 Привлечение новичков

### 3.15 AI-функции

#### 3.15.1 AI-тренер

Загрузи раздачу → получи разбор с рекомендациями. «Что бы сделал эксперт?» — AI показывает оптимальную линию. DD-анализ + LLM-комментарии.

#### 3.15.3 AI-авторазбор турнира

После турнира: ключевые моменты, лучшие розыгрыши, ошибки. Game Review (как Chess.com): точность торговли vs оптимальная, розыгрыш vs DD.

#### 3.15.4 AI-модерация

- Автоматическая детекция токсичности и спама в комментариях
- AI-помощник для поддержки (FAQ-бот)
- Детекция фейковых/координированных отзывов

#### 3.21.1 Кошелёк пользователя

Внутренний баланс на портале:
- **Пополнение:** банковская карта, СБП, ЮMoney
- **Вывод:** на банковскую карту
- **P2P-переводы:** оплата партнёру, тренеру, взнос за турнир
- **Полная история операций**

#### 3.21.2 Платежи

- Турнирные взносы — через кошелёк
- Оплата курсов/занятий — через кошелёк, тренер получает на баланс
- Оплата платного партнёра — эскроу (заморозка до завершения турнира)
- Донаты авторам
- Призовые выплаты

#### 3.21.3 Фонд развития бриджа

Общий фонд, формируемый из:
- Добровольные донаты
- % от комиссий маркетплейса
- Целевые сборы

**Расходование:**
- Поддержка клубов в регионах (аренда, оборудование)
- Мастер-классы, семинары
- Оплата тренерам в регионах
- Инвентарь (карты, борды, бидинг-боксы, скринки)
- Поездки юниоров на турниры
- Призовые фонды
- Развитие портала

**Прозрачность (до рубля!):**
- Публичный дашборд в реальном времени: баланс, поступления, расходы
- Каждая транзакция видна: дата, сумма, категория, получатель, основание, документы
- Ежемесячные и годовые отчёты (автогенерация)
- Выборный совет фонда — утверждает крупные расходы
- Аудит: любой участник может запросить детализацию

#### 3.21.4 Юридическая основа

- **Юрлицо:** НКО (АНО «Развитие бриджа») или ООО
- **Платёжный провайдер:** ЮKassa / Тинькофф / CloudPayments
- **54-ФЗ:** онлайн-кассы, чеки для платных услуг
- **152-ФЗ:** персональные данные
- **Публичная оферта** для пользователей

### 3.22 Реферальная система

- Реферальная ссылка «Приведи друга в бридж»

### 5.2 Обоснование ключевых решений

| Выбор | Альтернатива | Почему выбрали |
|-------|-------------|----------------|
| **Next.js** | SvelteKit, Rails, Python | Один язык (TS), SSR/SEO, крупнейшая экосистема, лучшая поддержка AI-кодинга (Claude/Cursor), нативная поддержка Telegram Mini App (React) |
| **tRPC** | REST, GraphQL | Type-safe от БД до UI без codegen/schema. GraphQL — overkill для 1-2 разработчиков |
| **Drizzle** | Prisma, TypeORM | Тонкий слой над SQL (~50KB vs 15MB Prisma), ближе к SQL-мышлению (Oracle-опыт), быстрее |
| **PostgreSQL** | MongoDB, MySQL | Стандарт для контент-платформ (Discourse, DEV.to). JSON-поддержка, full-text search, надёжность |
| **Self-hosted VPS** | Supabase Cloud, Vercel | Нет санкционных рисков, $0-20/мес, полный контроль |
| **Meilisearch** | Elasticsearch, Algolia | Лёгкий, быстрый, self-hosted, typo-tolerant, идеален для нашего масштаба |
| **shadcn/ui** | Material UI, Ant Design | Копируемые компоненты (не зависимость), Tailwind-native, красивые, кастомизируемые |
| **Rust (будущее)** | — | Точечно для DDS, AI-движок торговли, scoring, античит — где нужна производительность |

## 7. Интеграции

### 7.1 bridgesport.ru
- Парсинг календаря турниров
- Парсинг результатов
- Парсинг рейтингов (ежемесячно)

### 7.2 bridgemoscow.ru
- Импорт детальных результатов турниров
- Импорт раздач (LIN/PBN формат)

### 7.3 Telegram (второй фронтенд)

> Telegram — не просто «интеграция», а **второй фронтенд** портала. Веб — полная версия. Telegram — быстрый доступ, уведомления, действия на ходу.

#### 7.3.1 Авторизация
- Telegram Login Widget — вход в один клик
- Привязка Telegram-аккаунта к профилю

#### 7.3.2 Telegram-бот (@BridgeHubBot)

**Уведомления (портал → человек):**
- Ответили на комментарий
- Новый пост по подпискам
- Напоминание о турнире (за день / за час)
- Значимое изменение кармы
- Поиск партнёра на турнир, где ты свободен
- Финансы: пополнение, перевод, списание

**Действия из Telegram:**
- Ответить на комментарий
- Голосовать ⬆/⬇ inline-кнопками
- Записаться на турнир
- Откликнуться на поиск партнёра
- Решить задачу дня (квиз в боте)
- Команды: `/rating`, `/schedule`, `/next`

**Турнирный бот (для организаторов):**
- Напоминания участникам
- Сбор подтверждений: «Ты придёшь? ✅/❌»
- Live-обновления (результаты раундов)
- Сбор пост-турнирных отзывов кнопками

**Экстренная связь:**
- «Опаздываю на N минут» → уведомление организатору и партнёру
- «Ищу замену» → срочный поиск
- «Нужен судья на стол N» → вызов

#### 7.3.3 Telegram-канал
- Автопубликация: лучшие посты, задача дня, результаты, анонсы
- Воронка: канал → читает → переходит → регистрируется

#### 7.3.4 Синхронизация с клубными чатами
- Бот в группе клуба: новости, результаты, расписание
- Команды в чате: `/schedule`, `/results`, `/rating`, `/next`

#### 7.3.5 Telegram Mini App (Web App)
- Полноценное мини-приложение внутри Telegram
- Профиль игрока, задачник, маркетплейс партнёров, дашборд фонда
- «Попробуй бридж за 10 минут» — прямо в Telegram
- Нулевой барьер входа — люди уже в Telegram

#### 7.3.6 Инлайн-режим
- `@BridgeHubBot Шрамков` → карточка игрока
- `@BridgeHubBot турниры москва` → список турниров

#### 7.3.7 Обучение через бота
- Тренер задаёт домашку → ученик получает в Telegram
- Мини-уроки: карточки с теорией + квиз
- Ежедневная задача с разбором

## 8. Инфраструктура и безопасность

### 8.1 Безопасность

- **152-ФЗ** — согласие на обработку ПД, политика конфиденциальности
- **Право на удаление** — аккаунт и данные по запросу
- **PCI DSS** — при работе с картами (через провайдера)
- **2FA** — Telegram / TOTP
- **Шифрование** данных, регулярные бэкапы, аудит доступа

### 8.2 Доступность (Accessibility)

- Крупный шрифт, контрастность, простая навигация (много возрастных игроков)
- Режим «Крупный текст» — переключатель в настройках
- Адаптация для людей с ограниченными возможностями

### 8.3 Офлайн-режим (PWA)

- Турниры часто в подвалах с плохим интернетом
- Офлайн-доступ: задачник, конвенционная карта, расписание, профиль

### 8.4 Мультиязычность

- Английская версия интерфейса (бридж международный)
- Мультиязычные посты (RU + EN)

### 8.5 SEO

- Каждая статья, тег, профиль — индексируемая страница
- Structured data (Schema.org) для Google
- Sitemap, мета-теги, OG-теги для соцсетей
- Критично для популяризации

### 8.6 Мониторинг и DevOps

- Мониторинг доступности (uptime)
- Алерты при падении сервисов
- Автоматические бэкапы БД (ежедневно)
- Логирование и аудит доступа
- CI/CD: GitHub Actions → Docker → VPS

### 8.7 Аналитика портала (внутренняя)

- DAU / MAU / retention
- Воронка: зашёл → прочитал → зарегистрировался → первый комментарий → первый турнир
- A/B тестирование фич
- Метрики контента: популярные посты, теги, источники трафика

## 11. Монетизация

- **Freemium:** базовый доступ бесплатный (всё читаемо без регистрации)
- **Pro-подписка:** расширенный задачник, DD анализ, эксклюзивные серии, Bidding Explorer
- **Комиссия маркетплейса:** % от оплаты платного партнёра, курсов тренеров
- **Партнёрства:** с организаторами турниров, спонсорство
- **Реклама:** минимальная, только бриджевая (книги, клубы, турниры, инвентарь)
- **Донат авторам:** поддержка авторов (как Patreon)
- **Patron-система:** добровольные пожертвования, стена почёта, бейджи
